# 2.4 Robot Description and Runtime Configuration

## Robot Description

ROS 2 uses a **robot description** to define the structure, components, and capabilities of a robot.  
This description is typically written in **URDF (Unified Robot Description Format)** or **SDF (Simulation Description Format)**.

Key points:  
- Defines **links** (robot parts) and **joints** (connections)  
- Includes **sensors, actuators, and physical properties**  
- Provides **visual, collision, and inertial information**  

---

### Example Snippet (URDF)

```xml
<robot name="my_robot">
  <link name="base_link"/>
  <joint name="base_to_wheel" type="continuous">
    <parent link="base_link"/>
    <child link="wheel_link"/>
  </joint>
</robot>
```
This defines a simple robot with a base and a wheel joint.

---

## Runtime Configuration

Runtime configuration tells ROS 2 **how to run nodes, set parameters, and handle launch settings**.

- Nodes can have **parameters** (e.g., speed limits, sensor frequency)  
- Launch files (`.launch.py`) define **which nodes to start and in what order**  
- Enables **flexible deployment** for simulations or real robots

---

### Example Launch Command

```bash
ros2 launch my_robot_bringup bringup.launch.py
```
This starts all nodes defined in the launch file with proper configuration.
