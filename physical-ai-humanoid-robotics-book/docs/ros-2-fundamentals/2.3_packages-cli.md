# 2.3 Python Development and CLI Tools

## Developing ROS 2 with Python

Python is widely used to **write ROS 2 nodes**, interact with **topics, services, and actions**, and automate tasks.  
It’s readable, quick to develop, and integrates well with ROS 2’s libraries.

Example of a simple Python node:

```python
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class TalkerNode(Node):
    def __init__(self):
        super().__init__('talker')
        self.publisher_ = self.create_publisher(String, 'chatter', 10)
        self.timer = self.create_timer(1.0, self.publish_message)

    def publish_message(self):
        msg = String()
        msg.data = "Hello from ROS 2 Python node!"
        self.publisher_.publish(msg)
        self.get_logger().info(f"Published: {msg.data}")

def main(args=None):
    rclpy.init(args=args)
    node = TalkerNode()
    rclpy.spin(node)
    rclpy.shutdown()

if __name__ == "__main__":
    main()
```

## ROS 2 CLI Tools

ROS 2 provides **command-line tools** for managing nodes, topics, and packages:

- `ros2 node list` → List all running nodes  
- `ros2 topic list` → List all active topics  
- `ros2 topic echo /topic_name` → Display messages on a topic  
- `ros2 pkg create <package_name>` → Create a new ROS 2 package  

These tools help with **debugging, testing, and monitoring** ROS 2 systems in real time.

## Why Python & CLI Matter

- Python allows **quick, readable development** of ROS 2 nodes  
- CLI tools provide **real-time inspection and debugging**  
- Together, they make **robot development and testing faster and easier**
